# Project Configuration Preferences

## Architecture: Container / Presentational (Page + Screen) Pattern

UI screens use a two-layer split:

- **Page** (`packages/ui-library/src/components/*Page/`) — purely presentational. No hooks, no routing. Receives all data and callbacks via props. Fully Storybook-able.
- **Screen** (`apps/kartoffel-web/src/screens/`) — thin wiring layer. Calls `useNavigate`, `useUser`, and other app hooks, then renders the corresponding Page component with the results.

### Rules

- Pages live in `ui-library`, alongside components. They own the CSS and JSX.
- Screens live in the app. They own zero CSS and zero JSX — only hook calls and callbacks.
- Never import `react-router-dom` or app hooks inside a Page.
- Page props surface: pass only what the Page renders or calls (no raw `navigate` function, only `onBack`, `onContinueAnonymous`, etc.).
- Every Page must have a Storybook `.stories.tsx` file.

### Example

```tsx
// ui-library: OnboardingPage.tsx
interface OnboardingPageProps {
  onContinueAnonymous: () => void;
}
export function OnboardingPage({ onContinueAnonymous }: OnboardingPageProps) { ... }

// app: OnboardingScreen.tsx
export function OnboardingScreen() {
  const navigate = useNavigate();
  const { createAnonymousUser } = useUser();
  return (
    <OnboardingPage
      onContinueAnonymous={() => { createAnonymousUser(); navigate('/home'); }}
    />
  );
}
```

## Version Management

**Node.js and TypeScript versions should NOT be modified unless explicitly requested by the user.**

- Do not update `package.json` to change Node.js version requirements
- Do not modify TypeScript version in `package.json` dependencies or devDependencies
- Do not suggest or apply version upgrades for Node.js or TypeScript
- Respect the existing version constraints in the project

When suggesting changes to dependencies or project configuration, preserve the current Node.js and TypeScript versions unless the user specifically asks to upgrade or modify them.
